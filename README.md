# Московский паломник

Единая цифровая платформа для организации, планирования и сопровождения паломнических поездок по Москве и Московской области.

## Стек

- Laravel 9;
- PHP 8.1+;
- MySQL 8;
- Laravel Sanctum;
- Blade + Bootstrap 5;
- Яндекс.Карты JavaScript API 2.1;
- PWA Service Worker для выборочного офлайн-кэширования;
- database notifications Laravel;
- QR-коды в браузере и камера для проверки билетов.

## Календарь событий

Публичный календарь поддерживает:

- помесячную сетку и мобильный список;
- поиск по названию, описанию, месту и храму;
- фильтр по храму или паломническому объекту;
- фильтр по типу события;
- богослужения;
- престольные праздники;
- крестные ходы;
- паломнические поездки;
- лекции и встречи;
- семейные и молодёжные мероприятия;
- благотворительные события;
- многодневные события;
- события на весь день;
- отдельную страницу события;
- связь с храмом, маршрутом и организованной поездкой;
- адрес, координаты, контакты, вместимость и ссылку для регистрации;
- экспорт отдельного события в `.ics`.

Полный CRUD календаря доступен администратору:

```text
/admin/calendar
/admin/calendar/create
```

## QR-билеты

При создании бронирования автоматически формируются:

- читаемый код билета вида `MP-...`;
- случайный защищённый токен длиной 64 символа;
- QR-код с префиксом `MP-TICKET:`;
- страница билета для телефона;
- печатная версия;
- экспорт даты поездки в `.ics`.

Проверка билетов доступна администраторам, паломническим службам и представителям:

```text
/service/tickets/scanner
```

Сканер поддерживает:

- камеру смартфона или ноутбука;
- ручной ввод токена;
- просмотр владельца, поездки, статуса и количества участников;
- проверку отменённых и возвращённых билетов;
- отметку количества прибывших участников;
- фиксацию времени и сотрудника, выполнившего проверку;
- защиту от повторного использования QR-кода.

## Кабинет представителя храма

Доступен пользователям с ролью `object_editor` или `service_manager`:

```text
/service
/service/objects
/service/objects/{slug}/edit
/service/tickets/scanner
```

Представитель может:

- видеть только закреплённые за ним объекты;
- предлагать изменения описания, истории, адреса и координат;
- обновлять контакты и официальный сайт;
- обновлять расписание богослужений;
- указывать парковку и доступность;
- предлагать изменения списка святынь;
- загружать фотографии, видео, аудио и документы;
- отслеживать статус каждой заявки;
- проверять QR-билеты при посадке или регистрации.

Изменения не заменяют публичную карточку сразу. Администратор сначала проверяет заявку, затем одобряет и публикует её либо отклоняет с комментарием.

## Что ещё требует внешней интеграции

Создание бронирования и QR-билет работают, но реальное списание и возврат денег пока не подключены. Для завершения платёжного модуля необходимо выбрать провайдера и добавить его API, webhooks и онлайн-кассу.

Офлайн-режим сайта сохраняет выбранные карточки и локальные медиа. Полноценные офлайн-карты территории будут реализованы на стороне мобильного приложения, поскольку картографические тайлы имеют отдельные лицензионные ограничения.

Внутренние уведомления работают. Email и push-уведомления за день и за час требуют настройки почтового сервера, очередей, планировщика Laravel и push-провайдера.

Обсуждение совместного паломничества обновляется AJAX-запросами. WebSocket-сервер пока не подключён.

Тексты политики обработки персональных данных и правил сервиса являются техническими шаблонами. До публичного запуска заказчик должен указать юридическое лицо-оператора, контакты и сроки хранения, а затем утвердить документы с юристом.

## Установка

```bash
composer install
copy .env.example .env
php artisan key:generate
php artisan migrate --seed
php artisan storage:link
php artisan serve
```

Для Linux/macOS:

```bash
cp .env.example .env
```

Создайте базу MySQL `palomnik` и укажите доступы в `.env`.

Для карты добавьте:

```env
YANDEX_MAPS_API_KEY=ваш_ключ
```

Версию политики можно менять при публикации новой редакции:

```env
PRIVACY_POLICY_VERSION=1.0
```

Для локального администратора:

```env
ADMIN_NAME="Главный администратор"
ADMIN_EMAIL=admin@palomnik.local
ADMIN_PASSWORD=укажите_сложный_пароль
```

Затем:

```bash
php artisan config:clear
php artisan db:seed --class=AdminUserSeeder
```

## Обновление проекта

```bash
git pull origin main
composer install
composer dump-autoload
php artisan optimize:clear
php artisan migrate --seed
php artisan storage:link
php artisan test
```

После миграции существующие бронирования автоматически получают защищённые QR-токены.

Если `public/storage` уже существует, повторно создавать ссылку не требуется.
