# Мобильное приложение «Московский паломник»

Flutter-приложение для Android и iOS. Административная панель остаётся только в Laravel-сайте.

## Уже подключено

- регистрация и вход через Laravel Sanctum;
- безопасное хранение токена на устройстве;
- главная страница с объектами, маршрутами и событиями;
- каталог храмов и святынь;
- интерактивная Яндекс-карта через мобильное представление сайта;
- календарь событий;
- сообщество и «Паломничество вместе»;
- избранное;
- личный кабинет;
- статистика и достижения;
- персональные маршруты;
- уведомления;
- бронирования;
- отображение QR-билета;
- переход к построению маршрута в Яндекс Картах.

## Создание платформенных папок

Исходный Flutter-код хранится в `mobile`. Один раз выполните:

```bash
cd mobile
flutter create . --platforms=android,ios --org ru.mospalomnik
flutter pub get
```

Команда не должна заменять содержимое `lib` и `pubspec.yaml`. Перед её выполнением сохраните изменения в Git.

## Запуск с локальным Laravel

Android Emulator обращается к компьютеру через `10.0.2.2` — это уже значение по умолчанию:

```bash
flutter run
```

Для физического телефона укажите IP компьютера в локальной сети:

```bash
flutter run \
  --dart-define=API_BASE_URL=http://192.168.1.100:8000/api/v1 \
  --dart-define=SITE_BASE_URL=http://192.168.1.100:8000
```

Laravel при этом запускайте с доступом из локальной сети:

```bash
php artisan serve --host=0.0.0.0 --port=8000
```

Для боевого сервера:

```bash
flutter build apk --release \
  --dart-define=API_BASE_URL=https://example.ru/api/v1 \
  --dart-define=SITE_BASE_URL=https://example.ru
```

## Android

Добавьте разрешение в `android/app/src/main/AndroidManifest.xml`:

```xml
<uses-permission android:name="android.permission.INTERNET" />
```

Для тестирования по обычному HTTP в локальной сети временно разрешите cleartext-трафик у `<application>`:

```xml
android:usesCleartextTraffic="true"
```

В релизной сборке используйте только HTTPS.

## iOS

Для локального HTTP потребуется временное исключение ATS в `ios/Runner/Info.plist`. В публикации App Store сервер должен работать по HTTPS.

Минимальная целевая версия приложения: iOS 13 и Android 7/API 24, поскольку мобильная карта использует `webview_flutter`.
