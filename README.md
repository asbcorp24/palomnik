# Московский паломник

Единая цифровая платформа для организации, планирования и сопровождения паломнических поездок по Москве и Московской области.

## Стек

- Laravel 9;
- PHP 8.1+;
- MySQL 8;
- Laravel Sanctum;
- Blade + Bootstrap 5;
- Яндекс.Карты JavaScript API 2.1;
- PWA Service Worker для выборочного офлайн-кэширования.

## Публичный сайт

Основные страницы:

```text
/                         главная
/map                      интерактивная карта
/objects                  каталог храмов и святынь
/objects/{slug}           карточка объекта
/routes                   каталог маршрутов
/routes/{slug}            маршрут и ближайшие поездки
/community                сообщество
/community/{slug}         путевая заметка
/register                 регистрация паломника
/login                    вход паломника
/profile                  личный кабинет
/profile/favorites        избранное
/profile/bookings         бронирования и билеты
/profile/achievements     достижения
/profile/activity         отзывы, посещения, блог и медиа
/profile/settings         профиль, приватность и доступность
/my-routes                персональные маршруты
```

### Реализовано для пользователя

- регистрация и вход;
- хранение имени, email, телефона, аватара, даты рождения и интересов;
- настройки уведомлений, приватности, темы и размера шрифта;
- статистика профиля;
- персональные списки избранного;
- отметка о посещении с проверкой геолокации;
- автоматическая выдача достижений по подтверждённым посещениям;
- отзывы с модерацией;
- загрузка фото и видео с геометками;
- блог и путевые заметки с черновиками и модерацией;
- каталог маршрутов и расписание поездок;
- бронирование мест с контролем вместимости;
- код электронного билета;
- отмена бронирования;
- конструктор собственного маршрута из выбранных объектов;
- открытие пешего, автомобильного или общественного маршрута в Яндекс Картах;
- выборочное сохранение карточки объекта и медиа для просмотра без сети.

## Интерактивная карта

Поддерживаются:

- отображение храмов, монастырей, часовен и святых источников;
- фильтры по типу, викариатству, благочинию и святыне;
- поиск по названию, адресу и святыням;
- разные цвета маркеров;
- кластеризация объектов;
- определение местоположения;
- построение маршрута;
- стандартная карта, спутник и гибрид через переключатель Яндекс Карт.

Исторический слой XIX века требует отдельного лицензированного набора растровых тайлов и пока не подключён.

## Административная панель

```text
/admin/login
/admin
```

Реализованы:

- CRUD храмов и других объектов;
- типы объектов, викариатства, благочиния и святыни;
- фотографии, видео, аудиогиды, документы и схемы;
- CRUD маршрутов, точек маршрута и поездок;
- бронирования, статусы оплаты и билеты;
- достижения и квесты;
- подтверждение посещений;
- модерация отзывов, медиа и блога;
- управление пользователями;
- статистика платформы.

## Что ещё требует внешней интеграции

Создание бронирования уже работает, но реальное списание и возврат денег пока не подключены. Для завершения платежного модуля необходимо выбрать провайдера и добавить его API, webhooks и онлайн-кассу.

Офлайн-режим сайта сохраняет выбранные карточки и локальные медиа. Полноценные офлайн-карты территории будут реализованы на стороне мобильного приложения, поскольку картографические тайлы имеют отдельные лицензионные ограничения.

Push-уведомления за день и за час требуют настройки очередей, планировщика Laravel и push-провайдера.

## Установка

```bash
composer install
copy .env.example .env
php artisan key:generate
php artisan migrate --seed
php artisan storage:link
php artisan serve
```

Для Linux/macOS:

```bash
cp .env.example .env
```

Создайте базу MySQL `palomnik` и укажите доступы в `.env`.

Для карты добавьте:

```env
YANDEX_MAPS_API_KEY=ваш_ключ
```

Для локального администратора:

```env
ADMIN_NAME="Главный администратор"
ADMIN_EMAIL=admin@palomnik.local
ADMIN_PASSWORD=замените_пароль
```

Затем:

```bash
php artisan config:clear
php artisan db:seed --class=AdminUserSeeder
```

## Обновление проекта

```bash
git pull origin main
composer install
composer dump-autoload
php artisan optimize:clear
php artisan migrate --seed
php artisan storage:link
php artisan test
```

Если `public/storage` уже существует, повторно создавать ссылку не требуется.

## Тесты

```bash

```

Тесты используют SQLite в памяти и проверяют публичный каталог, админ-панель, регистрацию, избранное, отзывы, посещения, персональные маршруты и бронирования.

Разработка ведётся в основной ветке `main`.
